# 正则表达式基础教程

> 可以使用 [regex101.com](https://regex101.com/) 在线测试你的正则表达式。

## 限定符（Quantifiers）

限定符用于指定前面的字符（或组）出现的次数。

| 符号    | 含义                                   | 示例       | 匹配结果                             |
| :------ | :------------------------------------- | :--------- | :----------------------------------- |
| `?`     | 前面的字符出现 **0 次或 1** 次（可选） | `used?`    | `use`、`used`                        |
| `*`     | 前面的字符出现 **0 次或多次**          | `ab*c`     | `ac`、`abc`、`abbbc` 等              |
| `+`     | 前面的字符出现 **至少 1 次**           | `ab+c`     | `abc`、`abbc`、`abbbc` 等            |
| `{n}`   | 前面的字符出现 **恰好 n 次**           | `ab{5}c`   | 仅 `abbbbbc`                         |
| `{n,m}` | 前面的字符出现 **n 到 m 次**           | `ab{2,5}c` | `abbc`、`abbbc`、`abbbbc`、`abbbbbc` |
| `{n,}`  | 前面的字符出现 **至少 n 次**           | `ab{2,}c`  | `abbc`、`abbbc`、`abbbbc` 等         |

> **注意：** 限定符默认是**贪婪**的，会尽可能多地匹配字符。若想改为**懒惰**（尽可能少匹配），可在后面加 `?`，例如 `ab+?c`。



## " 或 " 运算符（OR Operator）

使用 `|` 表示逻辑“或”，可以匹配多个选项中的任意一个。

| 示例     | 匹配结果   |               |
| :----- | :----- | ------------- |
| `dog`  | `cat`  | `dog` 或 `cat` |
| `(ab)` | `(cd)` | `ab` 或 `cd`   |

> 括号 `()` 可用于分组，限定范围。

## 字符类（Character Classes）

字符类允许匹配一组字符中的**某一个**。

| 模式          | 含义                                          | 示例                    |
| :------------ | :-------------------------------------------- | :---------------------- |
| `[abc]`       | 匹配 `a`、`b` 或 `c` 中的任意一个             | `[aeiou]` 匹配元音      |
| `[a-c]`       | 同上，表示范围 `a` 到 `c`                     | `[0-9]` 匹配任意数字    |
| `[a-fA-F0-9]` | 匹配小写 a-f、大写 A-F 或 0-9（十六进制字符） |                         |
| `[^0-9]`      | **取反**，匹配除数字外的任意字符              | `[^a-z]` 匹配非小写字母 |

### 常用预定义字符类（元字符）

| 元字符 | 等价于               | 含义                               |
| :----- | :------------------- | :--------------------------------- |
| `\d`   | `[0-9]`              | 匹配一个数字                       |
| `\D`   | `[^0-9]`             | 匹配一个非数字                     |
| `\w`   | `[a-zA-Z0-9_]`       | 匹配字母、数字、下划线             |
| `\W`   | `[^a-zA-Z0-9_]`      | 匹配非单词字符                     |
| `\s`   | `[ \t\n\r\f]`        | 匹配空白字符（空格、制表、换行等） |
| `\S`   | `[^ \t\n\r\f]`       | 匹配非空白字符                     |
| `.`    | 除换行符外的任意字符 | 需注意模式修饰符的影响             |

> 在字符类内部，大部分元字符（如 `.`、`*`、`+`）会失去特殊含义，仅代表其字面值。但 `^` 在开头表示取反，`-` 表示范围，`\` 仍用于转义。



## 锚点与边界

| 符号 | 含义           | 示例                           |
| :--- | :------------- | :----------------------------- |
| `^`  | 匹配行首       | `^hello` 匹配以 hello 开头的行 |
| `$`  | 匹配行尾       | `world$` 匹配以 world 结尾的行 |
| `\b` | 匹配单词边界   | `\bword\b` 匹配完整单词 word   |
| `\B` | 匹配非单词边界 |                                |



## 贪婪与懒惰匹配

默认情况下，限定符（`*`、`+`、`{n,m}` 等）是**贪婪**的，会尽可能多地匹配字符。在其后加上 `?` 可变为**懒惰**模式，尽可能少地匹配。

| 模式    | 类型 | 示例文本          | 匹配结果                     |
| :------ | :--- | :---------------- | :--------------------------- |
| `<.+>`  | 贪婪 | `<div>text</div>` | 整个 `<div>text</div>`       |
| `<.+?>` | 懒惰 | `<div>text</div>` | `<div>` 和 `</div>` 分别匹配 |
